# Setup guide — Convi assignment

This project reads injury-claim data from **MongoDB**, builds a **Neo4j** graph (`graphIngest/`), and runs a **LangGraph agent** over that graph (`graphAgent/`). Optional MongoDB exploration lives in `analyzeData/`.

---

## Prerequisites

| Requirement | Notes |
|-------------|--------|
| **Python 3.11+** | 3.12 / 3.13 tested |
| **Git** | To clone the repo |
| **MongoDB access** | Remote `convi-assessment` URL (assessment credentials) |
| **Anthropic API key** | For `graphAgent` |
| **Neo4j** | Local graph store — see options below |

---

## 1. Clone and configure environment

```bash
git clone https://github.com/EldarAiki/convi-assingment.git
cd convi-assingment
```

Create your local env file from the template:

```bash
# Linux / macOS
cp .env.example .env

# Windows (PowerShell)
copy .env.example .env
```

Edit `assignment/.env` and set at minimum:

- `MONGODB_URL` — full connection string to `convi-assessment`
- `API_KEY_ANTHROPIC`
- `NEO4J_PASSWORD` — must match your Neo4j instance (see §2)

All subprojects load **`assignment/.env`** from the repo root (not per-folder `.env` files).

---

## 2. Neo4j — choose one option

The graph **is not stored in Git**. After Neo4j is running on a new machine, you must **ingest from MongoDB** (§4).

### Option A — Neo4j Desktop (GUI, good for exploration)

1. Install [Neo4j Desktop](https://neo4j.com/download/).
2. Create a local DBMS (e.g. name `convi-graph`).
3. Start it and note the **Bolt** port (default `7687`).
4. Set a password on first connect.
5. In `.env`:

   ```env
   NEO4J_URI=bolt://localhost:7687
   NEO4J_USER=neo4j
   NEO4J_PASSWORD=<your-desktop-password>
   NEO4J_DATABASE=neo4j
   ```

   **Note:** Desktop *instance name* ≠ database name. The database is usually `neo4j`.

### Option B — Docker (no Desktop app, recommended for portability)

Requires [Docker](https://www.docker.com/) only.

```bash
docker compose up -d
```

Default credentials (match `docker-compose.yml`):

```env
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=convi-dev-password
NEO4J_DATABASE=neo4j
```

Browser UI: http://localhost:7474  
Stop: `docker compose down` (data kept in Docker volume `neo4j_data`)

### Option C — Neo4j Aura (cloud, no local install)

1. Create a free [Neo4j Aura](https://neo4j.com/cloud/aura/) instance.
2. Copy the Aura **Bolt URI** and password into `.env`:

   ```env
   NEO4J_URI=neo4j+s://xxxx.databases.neo4j.io
   NEO4J_USER=neo4j
   NEO4J_PASSWORD=<aura-password>
   NEO4J_DATABASE=neo4j
   ```

## 3. Install Python dependencies

From the repo root:

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# Linux / macOS
source .venv/bin/activate

pip install -r requirements.txt
```

Or install per module only:

```bash
pip install -r graphIngest/requirements.txt   # ingest only
pip install -r graphAgent/requirements.txt    # agent only
```

---

## 4. Load the graph (required on every new Neo4j instance)

```bash
cd graphIngest
python main.py --dry-run          # optional: validate counts without Neo4j writes
python main.py --clear            # wipe Neo4j + full ingest from MongoDB
```

Expect ~70 cases, ~2,250 files, ~6k+ relationships. Report: `graphIngest/output/ingest_report.json`.

Verify in Neo4j Browser / Explore:

```cypher
:use neo4j
MATCH (n) RETURN labels(n)[0] AS label, count(*) AS count ORDER BY count DESC
```

---

## 5. Run the agent

```bash
cd graphAgent
python main.py -q "Find liability cases with overdue SLA"
python main.py -q "..." --html    # RTL-friendly HTML answer in output/
python main.py -q "..." --quiet   # suppress progress logs
```

Smoke-test Anthropic (from repo root):

```bash
python test_anthropic.py
python test_anthropic.py -p "Reply with exactly: API key works."
```

Agent trace: `graphAgent/output/agent_run.json`

## 6. Optional — data analysis / schema exploration

```bash
cd analyzeData
pip install -r requirements.txt
python profile/schema_profile.py
python explore/run_agent.py --question "How many cases are there?"
```

## Portability checklist

| Topic | In Git? | Action on new machine |
|-------|---------|------------------------|
| Source code | Yes | `git clone` |
| Secrets (`.env`) | **No** | Copy from `.env.example`, fill credentials |
| MongoDB data | **No** (remote) | Use same `MONGODB_URL` if you have access |
| Neo4j graph | **No** | Start Neo4j + run `graphIngest/main.py --clear` |
| Generated JSON reports | **No** (gitignored) | Regenerated by scripts |
| Python packages | **No** | `pip install -r requirements.txt` |
| Neo4j Desktop | **No** | Use Desktop, **Docker**, or **Aura** (§2) |

### Other things to watch

- **Hebrew in terminal** — PowerShell may render mixed RTL/LTR poorly; use `--html` for readable answers.
- **MongoDB IP allowlist** — Atlas may block new machines until their IP is allowed.
- **Neo4j empty after clone** — normal; ingest is mandatory.
- **Re-ingest after schema changes** — use `python main.py --clear` in `graphIngest/`.
- **Windows encoding** — `main.py` sets UTF-8 on stdout; use Windows Terminal if characters look wrong.

## Quick troubleshooting

| Problem | Check |
|---------|--------|
| `NEO4J_PASSWORD is not set` | Root `.env` exists and is filled |
| Agent finds 0 cases | Neo4j empty → run ingest `--clear` |
| `Authentication failed` (Anthropic) | `API_KEY_ANTHROPIC` in `.env` |
| MongoDB timeout | URL, network, Atlas IP allowlist |
| Wrong Neo4j DB in Browser | `:use neo4j` — match `NEO4J_DATABASE` |
| Old relationship counts | Re-run `python main.py --clear` after pull |
